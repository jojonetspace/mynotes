### 1.2 核心测试链路

```
平台下载 → 数据补全 → 编辑修改 → 发布 → 铺货复制 → 目标发布 → 源商品同步到目标
```

| 数据源 | 说明 |
|--------|------|
| 平台定时下载 | G.ItemDown → G.ItemQueue → G.ItemConvert（三阶段） |

## 3. 数据补全（EnsureListingData）测试

> 商品定时下载入库时可能仅含摘要数据。编辑/铺货/发布时需从平台 API 拉取完整详情，由 `ValidEnsureListingData` 完成。

### 3.1 补全触发条件（NeedEnsure）

| 条件 | NeedEnsure 结果 | 说明 |
|------|-----------------|------|
| shop_i_id 为空（草稿） | false | 未上传到平台的商品无需补全 |
| IsEnsuring=true | false | 正在补全中，避免重复 |
| plf_modified 为 null | false | 从未从平台下载过修改时间 |
| plf_modified > LastPlfModified | true | 平台有更新，本地未同步 |
| LastPlfModified 为 null 且 plf_modified 有值 | true | 首次需要补全 |
| plf_modified <= LastPlfModified | false | 已是最新数据 |

---

### TC-006：编辑页打开触发补全

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 打开商品编辑页 | LoadBySiId + ValidEnsureListingData |
| 2 | NeedEnsure()=true | 触发平台 API（DownByIds） |
| 3 | 按 shop_id 分组 | 同店铺商品合并为一次调用 |
| 4 | 补全成功 | data 更新，LastPlfModified 刷新 |
| 5 | 前端展示完整数据 | 类目/属性/图片/SKU/物流 |

---

### TC-007：补全异常场景

| 场景 | 预期行为 |
|------|----------|
| 店铺授权失效 | 不调 API，Error_="店铺授权已失效" |
| 店铺已禁用 | 不调 API，Error_="该店铺已禁用" |
| 平台商品已删除 | 返回空，标记下载失败 |
| API 限流/超时 | 内部重试或异常，标记失败 |
| 部分商品成功部分失败 | 分别处理，互不影响 |

---

### TC-008：已编辑商品不重复补全

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 商品已编辑保存 | LastPlfModified 已更新为 plf_modified |
| 2 | 再次打开编辑页 | NeedEnsure()=false（plf_modified <= LastPlfModified） |
| 3 | 不调用平台 API | 直接使用 l_data 展示 |

---

### TC-009：铺货前自动补全（列表页批量铺货）

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 列表页选源商品发起铺货 | ValidEnsureListingData 自动触发 |
| 2 | 源商品需补全（NeedEnsure=true） | 平台 API 拉取完整数据 |
| 3 | 补全成功 | 使用完整数据执行铺货 |
| 4 | 补全失败 | 铺货中止，返回错误 |

---

### TC-010：编辑页铺货不重复补全

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 编辑页打开（已触发补全） | 数据已完整，LastPlfModified 已更新 |
| 2 | 在编辑页发起铺货 | CopyShopItem 内仍调用 ValidEnsureListingData |
| 3 | NeedEnsure()=false | plf_modified <= LastPlfModified，不重复调用 API |

## 4. 商品下载流程测试

### TC-011：三阶段下载流水线

| # | 阶段 | 任务 | 预期结果 |
|---|------|------|----------|
| 1 | 阶段1 | G.ItemDown（按时间拉列表，写入队列） | 商品ID写入下载队列 |
| 2 | 阶段2 | G.ItemQueue（消费队列，拉详情写入中间表） | 写入 PlatformItem 中间表 |
| 3 | 阶段3 | G.ItemConvert（转换入库） | 入库 ShopItem 表 |

**验证点**：三阶段解耦、API 限流重试、SKU 匹配、重复下载更新而非新建

---

### TC-012：下载商品编辑发布（l_data 机制）

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 修改商品 | 写入 l_data（本地副本） |
| 2 | 保存不发布 | l_data 更新，listing_status=Inconsistent |
| 3 | 发布 | 使用 l_data 数据发布 |
| 4 | 发布成功 | ApplyLocalDataAndUpdate 提升主数据 |

---

### TC-013：下载覆盖与发布冲突

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 商品处于发布相关状态 | Uploading/PendingUpload/ScheduleUpload |
| 2 | 定时下载任务运行 | ShouldKeepOperationStatusOnDownload()=true |
| 3 | 下载保存 | 不覆盖 operation_status |

**验证点**：保护的状态包括 PendingUpload（待发布）、ScheduleUpload（定时发布）、Uploading（发布中），下载更新平台数据但不打断发布流程

---

### TC-014：发布失败与重试

| 步骤 | 操作 | 预期结果 |
|------|------|----------|
| 1 | 提交发布 | 创建任务 |
| 2 | 失败（限流） | error_count=1，Uploading（重试中 1/5） |
| 3 | error_count >= 5 时 | is_stopped=true，UploadFailure |

**验证点**：MaxRetryCount=5（error_count >= 5 即终止，实际最多重试4次）、SyncPublishStateToLocalData 保留中间状态、重试不重传已成功资源

### TC-041：上下架与状态流转

**状态矩阵**：

| 当前状态 | 允许操作 |
|----------|----------|
| Draft | 编辑、发布、删除 |
| OnSale | 编辑、发布、下架 |
| OffSale | 编辑、发布、上架、删除 |
| Deleted | 不允许发布 |

**operation_status 流转**：

| 当前 | 操作 | 目标 |
|------|------|------|
| Completed | 提交发布 | Uploading |
| Uploading | 成功 | Uploaded |
| Uploading | 可重试失败 | Uploading（重试中） |
| Uploading | 不可重试/重试耗尽 | UploadFailure |
| Incomplete | 补充字段 | Completed |
| UploadFailure | 重新发布 | Uploading |
