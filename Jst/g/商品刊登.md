项目中其他模块的并行处理模式总结
通过查看订单、物流、库存等模块的代码，项目中 Parallel.ForEach 的使用有一个统一的模式：

模式：并行内只做平台 API 调用，不做数据库操作
模块	并行内做什么	数据库操作在哪里
Shopee OrderDownService	调用 shopeePlatform.GetTrackingNumber() 获取物流单号	并行外统一处理
Shopee OrderQueueService	调用 shopeePlatform.GetTrackingNumber() 获取物流单号	并行外统一处理
Shopee/Lazada OrderProcessService	调用平台 API 发货（NormalOrderShip、Pack、ReadyToShip）	并行外 HandleWaitTodo 统一回写
ByQueueTaskBase	PageProcess() 处理数据（调平台 API）	并行结束后调 Save() 统一写库
DownloadByQueueBase	DownloadSingle() 调平台 API 下载	并行结束后调 SaveData() 统一写库
DrpStockCalcService	纯内存计算（CalcSingleDrpStock）	不涉及数据库

**日志：**
工作原理
后端添加日志：通过 CurrentUserContext.AddClientLog(log, isShow) 或 MasterService.AddClientLog(log, isShow) 添加
自动序列化：PageModelBase 的 Success()/Error() 等方法会自动调用 .AppendClientLog(GetClientLog())，将 ClientLog 列表附加到 JSON 响应的 client_log_ 字段
前端自动输出：$call 函数在收到响应后检查 rv.client_log_，如果有内容就 console.log(rv.client_log_) 输出到浏览器控制台
使用方式
在 Service 层中直接调用：

// 在 ShopItemService 中（继承自 MasterDABase，有 CurrentUserContext）
CurrentUserContext.AddClientLog("日志内容", true);

// 或通过 MasterService
MasterService.AddClientLog("日志内容", true);

项目中没有专门控制 ClientLog 输出的全局开关，但有以下机制可以利用：

ClientLogItem.show 字段：控制是否在 UI 上弹出提示（show=false 只输出到 console，不弹提示）
IsTempOpen 临时开关：可以通过后台配置页面（Admin/Config/TempOpen）按商家 ID 控制是否开启某个功能，适合做"是否输出性能日志到 ClientLog"的开关
