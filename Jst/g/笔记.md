**MasterService回填自增长主键**
场景一：InoutDA（出入库单 + 明细）

Write()
  └── WriteInoutNewList()        // 1. 写主表（Inout）
        ├── AutoWriteDb()        // 批量插入
        ├── LoadIoid()           // 2. 查询数据库取回 io_id（自增主键）
        └── IoidLoaded()         // 3. 将 io_id 回填到 io.Items[].io_id
  └── WriteInoutItemNewList()    // 4. 写从表（InoutItem），此时 io_id 已有值
关键代码在 IoidLoaded：

private void IoidLoaded(Inout io)
{
    if (io.Items != null)
    {
        foreach (var o in io.Items)
        {
            o.io_id = io.io_id;  // 回填外键
        }
    }
}
场景二：OrderDA（订单 + 明细/支付）

Write()
  └── WriteOrderNewList()
        ├── AutoWriteDb()        // 批量插入主表
        ├── LoadAndFillOid()     // 查询回填 o_id，并同步赋值给 Items[].o_id、Pays[].o_id
        └── AddEndStocks()
  └── WriteOrderItemNewList()    // 写从表，o_id 已有值
为什么用"查询回填"而不是直接拿插入返回值？
因为框架用的是 AutoWriteDb() 批量插入，批量插入后无法直接拿到每条记录的自增 ID，所以用业务唯一键（如 shop_id + so_id、o_id + type）反查数据库来取回自增主键，再回填到内存对象。

你的场景怎么做
按照这个模式，在你的 DA 的 Write() 里：

public override void Write()
{
    // 1. 写主表
    if (MasterList.Count > 0)
    {
        MasterList.DbConnectionAdapter = DbConnectionAdapter;
        MasterList.AutoWriteDb();
        
        // 2. 查询回填自增主键
        LoadAndFillMasterId(MasterList);
        
        // 3. 将主键赋值给从表的外键字段
        foreach (var master in MasterList)
        {
            foreach (var detail in master.Details)
            {
                detail.master_id = master.id;  // 回填外键
            }
        }
        
        MasterList.Clear();
    }
    
    // 4. 写从表（此时外键已有值）
    if (DetailList.Count > 0)
    {
        DetailList.DbConnectionAdapter = DbConnectionAdapter;
        DetailList.AutoWriteDb();
        DetailList.Clear();
    }
}
LoadAndFillMasterId 用业务唯一键反查，参考 LoadAndFillOid 的写法即可。
