# 商品刊登功能文档

## 一、功能概述

商品刊登是指将商品信息发布到电商平台(如Lazada、Shopee、TikTok、Facebook等)的完整流程,包括商品创建、编辑、复制、发布、状态管理等核心功能。

### 核心业务场景
- 创建新商品并发布到平台
- 编辑已有商品信息并同步到平台
- 复制商品到多个店铺
- 批量操作商品(上架/下架/更新)
- 下载平台商品到本地
- 商品状态管理和操作日志记录

## 三、核心业务流程

### 3.1 商品创建/编辑流程

```
用户操作 → 前端页面(ItemCreator.cshtml) → 后端API(OnPostSaveShopItem)
    ↓
ShopItemService.SaveShopItem()
    ↓
1. 保存ShopItem到数据库
2. 提取待D+上传的资源(图片/视频)
3. 判断operation_status:
   - IF PendingUpload(待发布):
     * IF 存在待D+处理资源 → 更新为PendingDPlus → return
     * ELSE 没有资源 → 直接调用PublishShopItem发布
   - ELSE 保存/暂存:
     * IF 存在待D+处理资源 → return
     * ELSE → return
    ↓
MasterService.WriteAll() → 提交数据库事务
```

### 3.2 商品发布流程

```
ShopItemService.SaveShopItem(ispublish=true)
    ↓
ShopItemPublishDataService.AddToWaitPublishQueue()
    ↓
创建ShopItemPublishData记录:
- biz_type = AddItem/UpdateItem
- status = Wait
- json_data = 商品数据JSON
    ↓
后台Task定时扫描:
- 取status=Wait且next_retry_time<=当前时间的记录
- 更新status=Uploading
    ↓
调用平台API发布:
ShopItemPublishService.Publish()
    ↓
根据平台类型调用对应实现:
- Lazada: LazadaShopItemPublishService
- Shopee: ShopeeShopItemPublishService
- TikTok: TikTokShopItemPublishService
- Facebook: FacebookShopItemPublishService
    ↓
发布结果处理:
- 成功: status=Success, operation_status=Uploaded
- 失败: status=Failure, error_count++, next_retry_time=计算重试时间
```

### 3.3 商品复制流程

```
用户选择源商品 + 目标店铺列表
    ↓
ShopItemService.CopyShopItem()
    ↓
1. 加载源商品完整数据(包含SKU)
2. 遍历目标店铺:
   For each targetShop:
     a. 创建新ShopItem对象
     b. 复制基础信息(title/pic/brand等)
     c. 清空平台相关字段(shop_i_id/shop_g_i_id等)
     d. 设置copy_from=源商品si_id
     e. 属性映射(如果跨平台/跨类目):
        - 产品属性映射(ProductProperties)
        - 销售属性映射(SalesProperties)
        - 使用向量相似度匹配
     f. 复制SKU列表
     g. 设置operation_status=Incomplete(待完善)
    ↓
3. 保存所有新商品到数据库
```

### 3.4 商品下载流程

```
用户触发下载操作
    ↓
ShopItemService.DownShopItem()
    ↓
1. 调用平台API获取商品数据
2. 解析平台返回的商品信息
3. 映射到ShopItem实体:
   - shop_i_id = 平台商品ID
   - shop_g_i_id = 平台全球商品ID
   - title/pic/price等基础信息
   - 平台特有属性存入data字段(JSON)
4. 保存/更新到数据库:
   - 如果shop_i_id已存在 → 更新
   - 如果不存在 → 新增
5. 下载商品SKU信息
6. 记录下载日志
```

## 五、关键业务规则

### 5.1 商品状态流转

```
Draft(草稿) → OnSale(上架) → OffSale(下架) → Deleted(删除)
     ↓            ↑              ↑
     └────────────┴──────────────┘
```

### 5.2 操作状态流转

```
Incomplete(待完善) → Completed(已完善) → PendingUpload(待发布)
                                              ↓
                                    ScheduleUpload(定时发布)
                                              ↓
                                      Uploading(发布中)
                                         ↙        ↘
                              Uploaded(已发布)  UploadFailure(已失败)
                                                      ↓
                                              PendingUpload(重试)
```

### 5.3 数据一致性规则

1. **商品与SKU关系**: 一个ShopItem包含多个ShopItemSku
2. **平台唯一性**: 同一店铺下,shop_i_id唯一
3. **内部编码映射**: i_id关联到Item表,sku_id关联到ItemSku表
4. **状态同步**: 商品上下架时,需同步更新所有SKU状态
5. **发布队列**: 发布操作通过队列异步处理,避免阻塞

### 5.4 商品复制规则

1. **同店铺同类目**: 直接复制所有属性
2. **跨店铺同平台**: 复制基础信息,清空平台ID
3. **跨平台**: 需要属性映射(使用向量相似度匹配)
4. **跨类目**: 需要重新选择类目和属性

## 八、注意事项

### 8.1 性能优化
- 商品列表采用分页加载
- 批量操作使用异步队列处理
- 图片/视频上传使用D+服务
- 数据库按商家ID分表(ShopItem_$1)

### 8.2 数据安全
- 所有操作记录审计日志(creator/modifier)
- 商品删除为逻辑删除(status=Deleted)
- 发布失败自动重试机制(error_count/next_retry_time)

### 8.3 扩展性
- 平台对接采用策略模式,易于新增平台
- 商品属性存储在data字段(JSON),支持平台特有属性
