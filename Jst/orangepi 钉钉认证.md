
## 到公司后的操作指南

### 方案一：Portal 认证（先试这个）

1. 插上网线
2. SSH 连到机器（通过 Tailscale 100.106.10.66，或者物理键盘）
3. 运行：
    
    bash
    
    ```
    portal-login
    ```
    
4. 脚本会自动检测认证页面，然后你选 `1`（w3m打开）
5. 在 w3m 中操作认证页面：
    - `Tab` 键切换到输入框/按钮
    - 输入内容后 `Enter` 提交
    - 如果是钉钉扫码：w3m 无法显示二维码，改用方案二

### 方案二：MAC 地址克隆（最稳妥，一劳永逸）

原理：公司的钉钉认证通常是绑定 MAC 地址的。你用公司里一台已经通过认证的电脑的 MAC 地址克隆到这台 Linux 上，网络设备会认为它是已认证的设备。

操作步骤：

1. 在公司找一台 已通过钉钉认证的 Windows 电脑，查看它的有线网卡 MAC：
    
    cmd
    
    ```
    ipconfig /all
    ```
    
    找到以太网适配器的"物理地址"，格式如 `AB-CD-EF-12-34-56`
    
2. SSH 到 Linux 机器，运行：
    
    bash
    
    ```
    mac-clone
    ```
    
    选 `1`，输入那台电脑的 MAC 地址（格式改为 `ab:cd:ef:12:34:56`）
    
3. 设置开机自动克隆：
    
    bash
    
    ```
    mac-clone-auto enable
    ```
    
4. 重要：克隆 MAC 后，那台被克隆的电脑必须断开有线网络或关机，否则同一网段会有 MAC 冲突。
    

> 最佳实践：克隆你自己在公司的工位电脑 MAC，然后平时用 WiFi，让 Linux 用有线。

### 方案三：如果认证是扫码且方案二不可行

到公司后先运行 `portal-login`，选 `4` 抓取认证页面源码给我看，我来帮你分析怎么用 curl 模拟认证。

---

总结已安装的工具：

|命令|用途|
|---|---|
|`portal-login`|一键检测并打开认证页面|
|`mac-clone`|克隆/恢复 MAC 地址|
|`mac-clone-auto enable`|设置开机自动克隆 MAC|
|`w3m`|文本浏览器，可操作 Web 表单|
|`lynx`|备用文本浏览器|
